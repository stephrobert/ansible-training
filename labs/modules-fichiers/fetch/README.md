## 🧠 Recap

🔗 [**Ansible fetch module**](https://blog.stephane-robert.info/docs/infra-as-code/gestion-de-configuration/ansible/modules/fichiers/module-fetch/)

`ansible.builtin.fetch:` is the **exact inverse of `copy:`**: it pulls a file
back from a managed node to the control node. It is used to collect **logs**,
back up **existing configurations** before migration, or retrieve **reports**
generated by the managed nodes.

`fetch:` handles two organization modes:

- **`flat: false`** (default): tree per host → `dest/<host>/<src>`.
- **`flat: true`**: single file in `dest:` (with `inventory_hostname`
  interpolation recommended).

## 🎯 Objectives

By the end of this lab, you will know how to:

1. **Understand** the default mode (tree per host) and its benefit.
2. **Use** `flat: true` with `inventory_hostname` interpolation for uniform
   collections.
3. **Fail** explicitly with `fail_on_missing: true` when the file must exist.
4. **Distinguish** `fetch:` (persistent file) from `slurp:` (read into memory).
5. **Diagnose** a relative `dest:` that resolves to the wrong place.

## 📚 Exercise 1 — Default mode (tree per host)

Create `lab.yml`:

```yaml
---
- name: Demo fetch arborescence
  hosts: webservers
  become: true
  tasks:
    - name: Backup sshd_config de chaque webserver
      ansible.builtin.fetch:
        src: /etc/ssh/sshd_config
        dest: collected/backup-default/
```

**Run**:

```bash
ansible-playbook labs/modules-fichiers/fetch/lab.yml
tree collected/backup-default/
```

🔍 **Observation**: structure created:

```text
collected/backup-default/
├── web1.lab/
│   └── etc/
│       └── ssh/
│           └── sshd_config
└── web2.lab/
    └── etc/
        └── ssh/
            └── sshd_config
```

**Advantages**:

- **No risk of overwriting** between hosts (each has its own subdirectory).
- **Preserves the absolute tree**, useful for audits that compare the same paths.

**Drawback**: significant tree depth. For a single file per host, it is a lot.
→ use `flat: true`.

## 📚 Exercise 2 — Flat mode (single file)

```yaml
- name: Recuperer un rapport audit (flat)
  ansible.builtin.fetch:
    src: /etc/os-release
    dest: "{{ inventory_dir }}/../collected/{{ inventory_hostname | regex_replace('\\.lab$', '') }}-os-release.txt"
    flat: true
```

**Run**:

```bash
ansible-playbook labs/modules-fichiers/fetch/lab.yml
ls collected/
```

🔍 **Observation**: flat structure:

```text
collected/
├── web1-os-release.txt
└── web2-os-release.txt
```

**Without `inventory_hostname` in `dest:`** on multi-host, **each host
overwrites the previous one**: a single file remains at the end.

**Rule**: `flat: true` + `dest:` interpolating `{{ inventory_hostname }}`.

## 📚 Exercise 3 — The relative `dest:` pitfall

```yaml
- name: Fetch avec dest relatif
  ansible.builtin.fetch:
    src: /etc/hostname
    dest: collected/hostname-{{ inventory_hostname }}
    flat: true
```

**Run**:

```bash
ansible-playbook labs/modules-fichiers/fetch/lab.yml
find . -name "hostname-*"
```

🔍 **Observation**: the file may land in `collected/` at the repo root... **OR**
in `labs/modules-fichiers/fetch/collected/`, depending on **where you run
ansible-playbook from**!

A relative `fetch: dest:` is resolved relative to **`playbook_dir`** (the
playbook's directory), **not** the execution cwd.

**Robust solution**: use **`{{ inventory_dir }}/../collected/`** to anchor the
path on the inventory (always stable):

```yaml
dest: "{{ inventory_dir }}/../collected/hostname-{{ inventory_hostname }}"
```

This is exactly the fix we applied to `inventory/hosts.yml` to resolve the SSH
key paths.

## 📚 Exercise 4 — `fail_on_missing: true`

By default, if `src:` does not exist, `fetch:` returns `ok: skipped` (no error).
On critical collections, you want to **fail** explicitly.

```yaml
- name: Recuperer un fichier qui n existe peut-etre pas
  ansible.builtin.fetch:
    src: /etc/myapp/license.key
    dest: collected/license-{{ inventory_hostname }}.key
    flat: true
    fail_on_missing: true
```

🔍 **Observation**:

- **Without `fail_on_missing: true`**: if `/etc/myapp/license.key` does not
  exist, the task is **silently skipped**. You discover the gap later.
- **With `fail_on_missing: true`**: the task **fails**, the PLAY RECAP shows
  `failed=N`, you see the problem **immediately**.

**Use case**: audits, critical backups, forensic log collections.

## 📚 Exercise 5 — `validate_checksum:` (integrity check)

`fetch:` automatically validates the **checksum** of the pulled-back file
(option `validate_checksum: true` by default). If the file changes during the
transfer, the task fails.

🔍 **Observation**: for very large files (GB), checksum computation can be slow.
**Disabling** is possible (`validate_checksum: false`) but should only be used
if you accept the risk of silent corruption.

## 📚 Exercise 6 — Multi-host audit pattern

```yaml
---
- name: Audit configurations critiques
  hosts: all
  become: true
  tasks:
    - name: Recuperer sshd_config de chaque host
      ansible.builtin.fetch:
        src: /etc/ssh/sshd_config
        dest: "{{ inventory_dir }}/../collected/audit-{{ ansible_date_time.date }}/sshd_config-{{ inventory_hostname }}"
        flat: true
        fail_on_missing: true

    - name: Recuperer ip a list
      ansible.builtin.shell: "ip -j a show > /tmp/ip-state-{{ inventory_hostname }}.json"
      changed_when: false

    - name: Rapatrier l etat reseau
      ansible.builtin.fetch:
        src: "/tmp/ip-state-{{ inventory_hostname }}.json"
        dest: "{{ inventory_dir }}/../collected/audit-{{ ansible_date_time.date }}/network-{{ inventory_hostname }}.json"
        flat: true
```

🔍 **Observation**: a complete **audit + centralized collection** pattern with
the day's timestamp.

## 📚 Exercise 7 — `fetch:` vs `slurp:`

`slurp:` (another Ansible module) **reads** a remote file and returns it as
**base64** in a variable, without writing on the control node.

```yaml
- name: Lire le contenu d un fichier sans le sauver
  ansible.builtin.slurp:
    src: /etc/hostname
  register: hostname_content

- name: Decoder et utiliser
  ansible.builtin.debug:
    msg: "Hostname raw : {{ hostname_content.content | b64decode }}"
```

| Case | Module |
|---|---|
| Save the file locally | `fetch:` |
| Read the content in memory for processing | `slurp:` |
| Compare the content before an action | `slurp:` then filter |
| Massive backup | `fetch:` |

🔍 **Observation**: `slurp:` is lighter (no disk write) but leaves no trace.
Prefer `fetch:` when you want an audit trail.

## 🔍 Observations to note

- **`fetch:`** = inverse of `copy:`, from the managed node to the control node.
- **Default mode**: tree per host (safe, never overwrites).
- **`flat: true`**: single file, **always interpolate `inventory_hostname`** on multi-host.
- **`fail_on_missing: true`** on critical collections (otherwise gaps are silent).
- **Relative `dest:`** is resolved from `playbook_dir`, prefer `{{ inventory_dir }}/../`.
- **`slurp:`** reads into memory (base64), **`fetch:`** saves to disk.

## 💡 Going further

- **`archive: + fetch:` pattern**: create a tarball on the managed node, then
  pull it back in a single operation. Useful to collect **several files**
  without 10 `fetch:`.
- **On-the-fly compression**: `fetch:` does not compress, go through `archive:`
  first (lab 35).
- **`add_host:` + `fetch:`**: dynamic pattern to collect from a host generated
  at runtime.
- **rsync via `synchronize:`** (ansible.posix collection): for bidirectional
  collections with minimal delta. More powerful but heavier.
