## 🧠 Recap

🔗 [**Build an Ansible collection**](https://blog.stephane-robert.info/docs/infra-as-code/gestion-de-configuration/ansible/collections/creer-collection-custom/)

A **custom collection** is the culmination of the collections track: package **several resources** (modules + roles + plugins + playbooks) under a single **versioned namespace**, distributable, versionable, testable.

This lab builds **`student.rhce.webapp`**: a minimalist collection with an **`nginx` role**, a **custom Python module** `webapp_healthcheck`, and an orchestration **playbook**. Validation via **`ansible-test sanity --docker`** green.

## 🎯 Objectives

By the end of this lab, you will know how to:

1. **Initialize** a collection with `ansible-galaxy collection init`.
2. Understand the **standard structure** generated (`plugins/`, `roles/`, `playbooks/`, `meta/`, `tests/`).
3. **Fill in `galaxy.yml`**: namespace, name, version, dependencies, tags.
4. **Add a role** in `roles/<role_name>/`.
5. **Add a custom Python module** in `plugins/modules/`.
6. **Build** the collection (`ansible-galaxy collection build`) → tarball.
7. **Test** with **`ansible-test sanity --docker default`**.

## 📚 Exercise 1 — Initialize the collection

```bash
cd labs/collections/creer-custom/
mkdir -p ansible_collections
ansible-galaxy collection init student.rhce.webapp \
  --init-path ansible_collections/
```

Output:

```text
- Collection student.rhce.webapp was created successfully
```

```bash
tree ansible_collections/student/rhce/webapp/ -L 2
```

🔍 **Observation**: `init` generates **the whole structure** of a compliant collection. There you find `galaxy.yml`, `README.md`, `LICENSE`, `meta/runtime.yml`, `plugins/{modules,filter,...}/`, `roles/`, `playbooks/`, `tests/`. No need to fill everything: keep only what you need.

## 📚 Exercise 2 — Fill in `galaxy.yml`

Edit `ansible_collections/student/rhce/webapp/galaxy.yml`:

```yaml
namespace: student
name: rhce_webapp
version: 1.0.0
readme: README.md
authors:
  - "Apprenant RHCE 2026 <student@example.com>"
description: "Collection lab 95 — déploie webapp avec nginx"
license:
  - GPL-3.0-or-later
tags:
  - linux
  - web
  - nginx
dependencies:
  ansible.posix: ">=2.0.0"
repository: https://github.com/student/rhce-webapp
documentation: https://github.com/student/rhce-webapp
issues: https://github.com/student/rhce-webapp/issues
build_ignore:
  - .github
  - .venv
  - tests/output
```

🔍 **Crucial observation**: **`tags:`** is **mandatory** to publish on Galaxy. Without it, the import fails silently. **`dependencies:`** declares the other required collections with semver constraints.

## 📚 Exercise 3 — Fill in `meta/runtime.yml`

Edit `ansible_collections/student/rhce/webapp/meta/runtime.yml`:

```yaml
---
requires_ansible: ">=2.18.0"
```

🔍 **Observation**: **`requires_ansible:`** is **mandatory** as soon as you add a custom Python module. Without it, `ansible-test sanity` fails. It is the declaration of the **minimum ansible-core threshold** on which the collection is guaranteed to work.

## 📚 Exercise 4 — Add an `nginx` role

Create `ansible_collections/student/rhce/webapp/roles/nginx/tasks/main.yml`:

```yaml
---
- name: Installer nginx
  ansible.builtin.dnf:
    name: nginx
    state: present

- name: Activer et démarrer nginx
  ansible.builtin.systemd_service:
    name: nginx
    state: started
    enabled: true
```

🔍 **Observation**: a role **in a collection** follows the same structure as a standalone role (`tasks/`, `handlers/`, `defaults/`, `templates/`...). The difference: you use it with its **FQCN**: `student.rhce.webapp.nginx`.

## 📚 Exercise 5 — Add a custom Python module

Create `ansible_collections/student/rhce/webapp/plugins/modules/webapp_healthcheck.py`:

```python
#!/usr/bin/python
# -*- coding: utf-8 -*-

from __future__ import absolute_import, division, print_function

__metaclass__ = type

DOCUMENTATION = r'''
---
module: webapp_healthcheck
short_description: Health check minimaliste d'une URL HTTP
version_added: "1.0.0"
description:
  - Vérifie qu'une URL HTTP répond avec un code 200.
options:
  url:
    description: URL à tester.
    required: true
    type: str
author:
  - "Apprenant RHCE 2026"
'''

EXAMPLES = r'''
- name: Vérifier que nginx répond
  student.rhce.webapp.webapp_healthcheck:
    url: "http://localhost:80"
'''

RETURN = r'''
status_code:
  description: Code HTTP retourné.
  type: int
  returned: always
'''

import urllib.request
from ansible.module_utils.basic import AnsibleModule


def run_module():
    module_args = dict(url=dict(type='str', required=True))
    module = AnsibleModule(argument_spec=module_args, supports_check_mode=True)

    url = module.params['url']
    try:
        with urllib.request.urlopen(url, timeout=5) as response:
            code = response.getcode()
        module.exit_json(changed=False, status_code=code)
    except Exception as exc:
        module.fail_json(msg=f"Health check KO: {exc}")


def main():
    run_module()


if __name__ == '__main__':
    main()
```

🔍 **Observation**: all Python modules follow this skeleton: `DOCUMENTATION`, `EXAMPLES`, `RETURN` in inline YAML + `AnsibleModule` from `module_utils`. **`ansible-test sanity`** checks these 3 sections.

## 📚 Exercise 6 — Add an orchestration playbook

Create `ansible_collections/student/rhce/webapp/playbooks/deploy.yml`:

```yaml
---
- name: Déployer la webapp avec la collection student.rhce.webapp
  hosts: webservers
  become: true
  tasks:
    - name: Appliquer le rôle nginx
      ansible.builtin.import_role:
        name: student.rhce.webapp.nginx     # ← role FQCN

    - name: Health check via le module custom
      student.rhce.webapp.webapp_healthcheck:    # ← module FQCN
        url: "http://localhost:80"
      register: hc

    - ansible.builtin.debug:
        var: hc.status_code
```

🔍 **Observation**: from outside the collection, you call modules and roles with **their full FQCN**. Inside a collection, you can use short names (but this is discouraged for readability).

## 🔍 Observations to note

- **`ansible-galaxy collection init`** generates the complete compliant structure.
- **`galaxy.yml`**: namespace, name, semver version, **`tags:` mandatory**, dependencies.
- **`meta/runtime.yml`**: **`requires_ansible:`** mandatory as soon as you have a Python module.
- **Python module**: `DOCUMENTATION` + `EXAMPLES` + `RETURN` in YAML + `AnsibleModule`.
- **FQCN** from outside the collection: `namespace.collection.plugin`.
- **`ansible-galaxy collection build`** produces a publishable tarball.
- **`ansible-test sanity --docker default`** validates doc + types + FQCN + PEP8.

## 🤔 Reflection questions

1. Why must a **`namespace`** on Galaxy be **reserved** before publishing? (Hint: avoid **typosquatting attacks**).

2. What happens if you publish **without `tags:`** in `galaxy.yml`? (Hint: Galaxy import KO).

3. What is **`build_ignore:`** used for? (Hint: exclude `.git/`, `.venv/`, test secrets from the tarball).

4. How do you handle a **breaking change** on the module? (Hint: major bump + `meta/runtime.yml plugin_routing.deprecation`).
