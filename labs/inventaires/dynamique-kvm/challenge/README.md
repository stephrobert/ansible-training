# 🎯 Challenge: target only the running VMs of the lab

You have seen the libvirt plugin in action. The challenge is to **combine** the dynamic groups (`state_running`) and the static ones (`lab_vms`) to target **precisely** the lab VMs that are running.

## ✅ Objective

Write `solution.yml` that:

1. Targets **`hosts: lab_vms:&state_running`** (intersection: lab VMs AND running).
2. Lays down a marker file `/tmp/lab57-mark-{{ inventory_hostname }}.txt` that contains:

   ```text
   VM <nom> running - inventaire dynamique libvirt OK
   ```

   The `<nom>` is that of the **libvirt domain**: `web1.lab`, not `web1`. The
   marker of `web1.lab` is therefore called `/tmp/lab57-mark-web1.lab.txt`.

## 🧩 Instructions

Skeleton to complete:

```yaml
---
- name: "Challenge : cibler les VMs running du lab via plugin libvirt"
  hosts: ???                       # intersection: static group 'lab_vms' AND dynamic group 'state_running'
  gather_facts: false
  tasks:
    - name: Poser le marqueur
      ansible.builtin.copy:
        dest: ???                  # /tmp/lab57-mark-<inventory_hostname>.txt
        content: "VM {{ ??? }} {{ ??? }} - inventaire dynamique libvirt OK\n"
        mode: "0644"
```

> 💡 **Traps**:
>
> - **Inventory = folder**, not a YAML file. Ansible reads all the `*.yml` of
>   `inventory/`, plus `group_vars/` and `host_vars/`, and merges. Run with
>   `-i inventory/` (with the trailing slash).
> - **The name is that of the libvirt domain.** `inventory_hostname: name`
>   reports `web1.lab`, what `virsh list --all` shows. A playbook or a filter
>   written on `web1` matches **nothing**, and Ansible does not complain.
> - **State groups**: `state_running`, `state_shutoff`, and so on. They come
>   from the `keyed_groups` of `01-libvirt.yml`, computed from `info.state`
>   that the plugin reports. No static file declares them.
> - **Do not hardcode `running`** in the content of the marker: the marker must
>   **prove** that you read the state reported by the plugin. The variable is
>   `info.state`.
> - **`&` pattern**: intersection. `lab_vms:&state_running` = "in `lab_vms` AND
>   in `state_running`". Both halves count: `state_running` alone would go and
>   write on all the powered-on VMs of your machine, including those that have
>   nothing to do with the training.
> - **Plugin cache**: if you shut off a VM then rerun the playbook, the
>   inventory reflects the state **at run time** (no cache by default).

Run the solution from the lab folder:

```bash
cd labs/inventaires/dynamique-kvm/
ansible-playbook -i inventory/ challenge/solution.yml
```

Check the markers on the 4 lab VMs. The connection goes through the `ssh_config`
generated by dsoxlab: no IP, no user to know.

```bash
SSH_CFG=~/.cache/dsoxlab/ansible-training/ssh_config
for h in control-node.lab web1.lab web2.lab db1.lab; do
  ssh -F "$SSH_CFG" "$h" "cat /tmp/lab57-mark-${h}.txt"
done
```

Expected output (4 lines):

```text
VM control-node.lab running - inventaire dynamique libvirt OK
VM web1.lab running - inventaire dynamique libvirt OK
VM web2.lab running - inventaire dynamique libvirt OK
VM db1.lab running - inventaire dynamique libvirt OK
```

## 🧪 Validation

The pytest tests check that the **4 markers** exist with the right content,
that the pattern resolves **only** the 4 lab VMs (no foreign VM touched), and
that the playbook is **idempotent**.

```bash
pytest -v challenge/tests/
```

## 🚀 Going further

- Shut off `web2.lab` (`virsh -c qemu:///system shutdown web2.lab`), rerun the
  playbook: only 3 markers are created, `web2.lab` is no longer in
  `state_running`. Power it back on afterwards (`virsh -c qemu:///system start web2.lab`).
- Check that a powered-on VM **outside the lab** is never touched:
  `ansible -i inventory/ 'lab_vms:&state_running' --list-hosts` must always
  return exactly 4 hosts, whatever else runs on your machine. Compare with
  `ansible -i inventory/ 'all' --list-hosts`.
- Modify `01-libvirt.yml` to add a `production` group that contains
  `web1.lab`, `web2.lab` and `db1.lab` but **not** `control-node.lab`. Test
  with `ansible -i inventory/ production -m ansible.builtin.ping`.
- Move `ansible_connection: ssh` from `host_vars/` to `group_vars/lab_vms.yml`
  and rerun: watch Ansible fall back on `libvirt_qemu` and fail. This is
  precedence, live.
