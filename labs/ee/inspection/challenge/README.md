# 🎯 Challenge : compare 3 Execution Environments, with proof

## ✅ Mission

Write `inspect.sh` (delivered as a skeleton at the lab root), run it,
and produce a factual comparison report.

| Deliverable | Expectation |
| --- | --- |
| `inspect.sh` | completed (no more `???`), executable, valid bash syntax. Inspects the 3 EEs `creator-ee`, `awx-ee`, `community-ee-minimal` with podman and ansible-navigator |
| `inspect-output/comparison.md` | generated by YOUR script: a Markdown table with, for each of the 3 EEs, the observed ansible-core version and the image size |

The report must come from an actual run of the script (`./inspect.sh`),
not from a copy-paste: the observed versions change with the
`latest` tags, which is precisely the point of the inspection.

## 🧩 Stuck?

```bash
dsoxlab hint ee-inspection
```

Hints are progressive and **cost points**: the first one points you in the
right direction, the last one unblocks you.

## 🚀 Launch

```bash
cd labs/ee/inspection/
./inspect.sh
cat inspect-output/comparison.md
```

## 📓 Command log

Record in `challenge/solution.sh` the commands you ran. This log
must exist for pytest to run:

```bash
chmod +x challenge/solution.sh
```

## 🧪 Validation

```bash
pytest -v labs/ee/inspection/challenge/tests/
```

Pytest checks your script (real bash syntax, tools used, 3 EEs
covered) and the factual content of the generated report. It does not pull the
images itself (several GB): it is your run that produces the
report.

## 🧹 Reset

```bash
dsoxlab clean ee-inspection
```
